# 前后端启动方法

本文只记录本地开发启动步骤。环境安装细节见 `docs/5-COLMAP与Nerfstudio安装指南.md`。

## 1. 启动后端

从项目根目录执行：

```powershell
cd D:\Projects\3dgs-app
```

如果只是验证前后端联通，不跑真实 3DGS 训练，使用 mock 模式：

```powershell
$env:BACKEND_PIPELINE_MOCK = "true"
backend\.venv\Scripts\python -m uvicorn backend.app.main:app --host 0.0.0.0 --port 8000 --reload
```

如果要跑真实链路，先激活 `nerfstudio / COLMAP / tiny-cuda-nn` 所在的 conda 环境，再启动后端：

```powershell
conda activate 3dgs_app
cd D:\Projects\3dgs-app
$env:BACKEND_PIPELINE_MOCK = "false"
$env:TRAINING_QUALITY_PROFILE = "balanced"
$env:TRAINING_MAX_STEPS = "7000"
backend\.venv\Scripts\python -m uvicorn backend.app.main:app --host 0.0.0.0 --port 8000 --reload
```

现在创建重建任务只会上传视频并生成 `task_id`，不会自动开始训练。进入任务状态页后，需要在“训练设置”中选择质量档位和训练轮数，再手动点击“开始训练流水线”。流水线运行中可在状态页点击“终止流水线”，后端会结束对应进程树并把任务标记为 `cancelled`。

当前本地 COLMAP 的位姿恢复和 BA 按 CPU 跑。虽然 `D:\colmap-x64-windows-cuda\bin\colmap.exe` 是 CUDA build，但它链接的 Ceres 没有 CUDA/cuDSS 支持；强行传 GPU BA 参数也会回退到 CPU solver。详见 `docs/12-COLMAP位姿恢复与BA无法使用GPU遗留问题报告.md`。

启动后先在电脑浏览器打开：

```text
http://127.0.0.1:8000/health
```

如果用手机真机调试，还要用手机浏览器打开对应的电脑地址。具体地址取决于下面第 2 节的连接方式。

如果手机打不开，优先检查后端是否使用 `--host 0.0.0.0`、Windows 防火墙是否放行 `8000` 端口，以及手机是否能访问电脑所在网络。远程办公场景下，局域网 IP 可能一直加载，这时优先改用第 2.3 节的电脑 Tailscale IP。

## 2. 启动 Flutter 前端

另开一个终端：

```powershell
cd D:\Projects\3dgs-app\mobile_app
flutter pub get
```

### 2.1 Android 模拟器

Android 模拟器访问宿主机后端使用 `10.0.2.2`：

```powershell
flutter run --dart-define=API_BASE_URL=http://10.0.2.2:8000
```

模拟器内可用浏览器验证：

```text
http://10.0.2.2:8000/health
```

### 2.2 真机：数据线或同局域网连接

如果手机和电脑在同一个 Wi-Fi / 局域网，或手机通过数据线调试但网络仍走同一个局域网，App 的后端地址使用电脑的局域网 IPv4。

在电脑上查看 IPv4：

```powershell
ipconfig
```

找到当前联网网卡的 IPv4，例如 `WLAN` 下的：

```text
192.168.5.127
```

先用手机浏览器验证：

```text
http://192.168.5.127:8000/health
```

确认能打开后启动 Flutter：

```powershell
flutter run --dart-define=API_BASE_URL=http://192.168.5.127:8000
```

不要在真机上使用 `10.0.2.2`，它只适用于 Android 模拟器。

### 2.3 真机：Tailscale 连接

如果远程办公时用 Tailscale 连接手机和电脑，推荐使用这种方式。App 的后端地址应使用“电脑的 Tailscale IP”，不是手机的 ADB IP。

可以在电脑上用 `ipconfig` 找到 `Unknown adapter Tailscale` 下的 IPv4，例如：

```text
100.95.147.1
```

也可以在有权限的终端中执行：

```powershell
tailscale ip -4
```

先用手机浏览器验证：

```text
http://100.95.147.1:8000/health
```

如果 `http://192.168.5.127:8000/health` 这类局域网地址在手机上一直加载，但电脑本机能打开，通常说明手机到电脑的局域网路径不通；改用电脑 Tailscale IP，例如 `http://100.95.147.1:8000/health`。

确认能打开后启动 Flutter：

```powershell
flutter run --dart-define=API_BASE_URL=http://100.95.147.1:8000
```

如果通过 Tailscale 做无线 ADB，命令可能类似：

```powershell
D:\Android\Sdk\platform-tools\adb.exe pair 100.121.83.102:41103
D:\Android\Sdk\platform-tools\adb.exe connect 100.121.83.102:45495
```

这里的 `100.121.83.102:45495` 是“手机的 Tailscale IP + ADB 调试端口”，只给 `adb / flutter run` 连接设备使用。`API_BASE_URL` 必须填写“电脑的 Tailscale IP + 后端端口”，例如 `http://100.95.147.1:8000`。

## 3. 常用入口

- 创建重建任务：打开 App 首页，填写商品信息并选择视频；创建后在训练设置中手动启动流水线。
- 查看已有模型：打开 App 首页，点击“查看已训练模型”。
- 后端健康检查：`GET /health`。
- 任务接口：`GET /api/v1/reconstructions`。
- 查看器入口：后端返回的 `viewer_url`，最终会加载 `/viewer/index.html`。

## 4. 常见问题

- 真机上传超时：不要用 `10.0.2.2`。同局域网真机用电脑局域网 IPv4；Tailscale 真机用电脑 Tailscale IP。
- 远程办公手机访问局域网 IP 一直加载：优先改用电脑 Tailscale IP，例如 `http://100.95.147.1:8000`。
- Tailscale 调试混淆：`adb connect` 后面的 IP 是手机 ADB 地址，不是后端地址。App 的 `API_BASE_URL` 要指向电脑。
- 真实训练找不到 `ns-train`：后端启动前先 `conda activate 3dgs_app`。
- 真实训练找不到 `colmap`：确认 `D:\colmap-x64-windows-cuda` 或 `D:\colmap-x64-windows-cuda\bin` 已进入系统 `Path`，并重新打开终端。
- 查看器没更新：重启后端，并重新 `flutter run` 或至少 `hot restart`，因为 WebView 会缓存静态资源。
